# AIX 脉盔项目审阅与下一步建议

> 更新日期：2026-07-13

## 当前定位

工程已收敛为“ESP32-S3 压力采集 + OV5640 本地健康采帧 + PC 状态监测”原型。此前的 PC/手机摄像头、OpenCV 光流、模拟目标检测、风险融合、语音及气囊模拟执行链路已移除。

```text
XGZP6847A -> pressure_sensor -> pressure -> 上位机
OV5640 -> camera_local -> camera_status -> 上位机状态卡
```

当前上位机不接收相机 JPEG；它只在串口连接后通过 `camera_status` 判断 OV5640 是否在持续采帧。状态正常只代表相机健康，不代表已经识别道路目标。

## 架构评价

- 当前固件职责明确：压力传感器与相机健康采帧相互独立。
- 上位机不再承担视觉判断，避免 PC、手机和 ESP32-S3 三套视觉来源混用。
- `motion` 协议和面板继续作为未来扩展预留；风险与气囊仅保留静态 UI 位置。
- `camera_local` 与未来 ESP32-S3 视觉模型之间保持边界：采帧模块负责帧与状态，模型模块将负责后续推理。

## 已知边界

- OV5640 代码已实现，尚未完成实机接线、五分钟连续采帧和故障恢复验收。
- 未实现目标检测、距离估算、TTC、语音或真实气囊执行。
- 未确认 PSRAM 前，维持 DRAM 单缓冲，不应贸然启用高分辨率或双缓冲。

## 推荐开发顺序

1. 按硬件接线文档完成 OV5640 实机稳定采帧验收。
2. 在 ESP32-S3 增加独立视觉模型模块，消费 `camera_local` 帧，不向上位机传 JPEG。
3. 模型稳定后，再重新定义检测结果协议、风险策略和执行安全状态机。
4. 最后接入真实泵阀前完成限压、泄气、超时和人工急停台架验证。
