# 气泵、三通电磁阀与 MPU6050 接线说明

适用对象：ESP32-S3-DevKitC-1、两个 3.3V 高电平有效 MOSFET 驱动板、两个 SS54、额定 DC 4.5–6V 的泵/阀负载，以及图示带 5V LDO 的 MPU6050 小板。

## 先确认安全边界

- 泵和电磁阀由**独立电池组/外部 5–6V 电源**供电；禁止从 ESP32 开发板 5V 引脚给它们供电。
- 两个 MOSFET 模块的信号地、ESP32 GND、外部电源负极必须共地。
- 每个感性负载各并联一个 SS54。二极管条纹端（阴极）接该负载的正端，另一端（阳极）接负载负端。接反会在通电时造成短路。
- 未刷入并确认本功能固件前，不接泵/阀负载；首次只测 MOSFET 输出端，确认 GPIO40/41 上电都是低电平。
- 旧压力模块的 `180 kPa` 仅是传感器诊断阈值，不是气囊控制阈值。气动控制的硬上限来自保存的标定配置，固件限制在 `<=40 kPa`。

## 电源与两块 MOSFET 板

| 连接对象 | 气泵 MOSFET 板 | 电磁阀 MOSFET 板 |
|---|---|---|
| 控制输入 `IN/IO` | ESP32 `GPIO40` | ESP32 `GPIO41` |
| 信号地 `GND` | ESP32 `GND` | ESP32 `GND` |
| 电源正 `VIN+/DC+` | 电池组正极 | 电池组正极 |
| 电源负 `VIN-/DC-` | 电池组负极 | 电池组负极 |
| 输出正 `OUT+` | 气泵正极 | 电磁阀正极 |
| 输出负 `OUT-` | 气泵负极 | 电磁阀负极 |
| SS54 | 条纹端接 `OUT+`，另一端接 `OUT-` | 条纹端接 `OUT+`，另一端接 `OUT-` |

两个模块都按上表完整连接；不要因为外部电源已经接在 `VIN-` 上就省掉 ESP32 与模块之间的信号地。

```text
                 外部 5–6V 电池组
              + ─────────┬──────────> 两块 MOSFET 的 VIN+/DC+
              - ───┬─────┴──────────> 两块 MOSFET 的 VIN-/DC-
                   └────────────────> ESP32 GND、两块 MOSFET 信号 GND

ESP32 GPIO40 ───────────────────────> 气泵 MOSFET IN/IO
ESP32 GPIO41 ───────────────────────> 电磁阀 MOSFET IN/IO

气泵：       OUT+ ── 负载 ── OUT-
                         │      │
SS54：          条纹端 ──┘      └── 无条纹端

电磁阀：     OUT+ ── 负载 ── OUT-
                         │      │
SS54：          条纹端 ──┘      └── 无条纹端
```

上图的 SS54 是反向并联在负载两端：正常工作时不导通，断开 MOSFET 时吸收线圈反电动势。

## 气路

图示三通阀的内部关系是：断电 `1 ↔ 3`、`2` 封闭；通电 `1 ↔ 2`、`3` 封闭。因此按下表接气管，软件断电即泄压。

| 阀口 | 接到 | 结果 |
|---|---|---|
| `1` | 气囊，同时接压力传感器取压三通 | 系统公共气腔 |
| `2` | 气泵出气口 | 阀通电时充气 |
| `3` | 排气口/消音器 | 阀断电时自动泄压 |

气泵进气口保持通向洁净空气。气阀 `GPIO41=低` 时为断电泄压；`GPIO41=高` 时才连接泵到气囊。软件在故障、急停、压力无效或压力过期时同时关闭泵和阀，利用该气路泄压。

## MPU6050 小板

下表仅适用于所给图片中“VCC 5V 输入、板载 LDO 和 I2C 3.3V 上拉”的 MPU6050 模块。若是裸 MPU6050 或未知模块，先确认其 I2C 电平，再接线。

| MPU6050 引脚 | ESP32-S3-DevKitC-1 | 说明 |
|---|---|---|
| `VCC` | 开发板 `5V` | 仅给 MPU 小板供电，不给泵/阀供电 |
| `GND` | ESP32 `GND` | 与外部电源负极共地 |
| `SDA` | `GPIO2` | I2C 数据，地址默认 `0x68` |
| `SCL` | `GPIO6` | I2C 时钟 |
| `INT` | `GPIO39` | 数据就绪中断 |
| `AD0` | 悬空或接 GND | 使用 `0x68`；不要接高电平变成 `0x69` |
| `XDA`、`XCL` | 不接 | 本方案不使用辅助 I2C |

## 上电顺序与验收

1. **断开泵和阀负载**，接好 ESP32、MOSFET 信号线、共地和 MPU6050。
2. 编译时先只启用 `CONFIG_AIX_ENABLE_PNEUMATIC_CONTROL=y`；保持 `CONFIG_AIX_ENABLE_PNEUMATIC_AUTOMATIC=n`。自动模式默认关闭且不能由上位机页面打开。
3. 上电后用万用表测两块 MOSFET 的 `OUT-`：没有手动命令时应为关断，电磁阀应处于断电泄压状态。
4. 确认串口可见 `motion` v2、`pressure` 和 `pneumatic_status`。MPU6050 静置约 2 秒完成 200 个样本的校准。
5. 接入电磁阀，先在“气动标定”页点“泄压”，再使用一次 `200 ms 充气脉冲` 验证 GPIO40 和输出方向；确认后才接气囊。
6. 设置并保存受限的标定值，例如目标 `2.0 kPa`、硬上限 `4.0 kPa`、最大充气 `2000 ms`。保存成功才会写入设备 NVS。
7. 测量泵的实际启动/堵转电流后，再确定电池、线径和电池正极保险丝额定值。电源持续电流能力必须覆盖两负载同时工作的峰值并留有余量。

## 当前固件行为

- 手动页只允许固定 200ms 充气脉冲、泄压、急停、故障复位和保存限制。
- `pressure` 无效或超过 200ms 未刷新、超过保存的硬上限、充气超时、保压超时、急停都会锁存 `fault_vent`：泵关、阀断电泄压。
- 自动条件是“已保存标定”且“固件编译时明确开启自动模式”，再满足新鲜视觉 `HIGH/CRITICAL` 或 MPU 的 `impact/rapid_tilt` 任一事件。默认构建不会自动充气。
